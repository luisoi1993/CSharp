SaveInstantInvoiceAsync()
│
├── PURPOSE (Objetivo)
│   : Crear una factura emitida instantánea completa
│   : Validar todos los requisitos fiscales, legales y de negocio
│   : Persistir factura + vencimientos + conceptos + impuestos en una transacción
│   : Devolver la entidad factura creada (sin número, se asigna después)
│
│   Resultado final:
│   ├── InvoiceIssuedDTE (factura persistida)
│   ├── BillIssued (vencimientos asociados)
│   ├── BillConcept (conceptos/líneas de factura)
│   ├── InvoiceTax (desglose impositivo)
│   └── InvoiceIssuedPersonData (snapshot datos cliente)
│
│   Atributo transaccional:
│   └── [Transactional(REQUIRED_NEW, DBContext)]
│       : Nueva transacción independiente
│       : Si falla cualquier paso → rollback total
│       : Si todo OK → commit atómico
│
│
├── INPUT (Entrada)
│   │
│   ├── InstantInvoiceDTO instantInvoiceDTO
│   │   : Datos de la factura a crear
│   │
│   │   Campos importantes:
│   │   ├── CompanyUID
│   │   │   : UID de la empresa emisora
│   │   ├── ParticipantUID
│   │   │   : UID del cliente receptor
│   │   ├── SerieID
│   │   │   : ID de la serie seleccionada
│   │   ├── DraftInvoiceIssuedID
│   │   │   : ID del borrador (opcional, null si no viene de borrador)
│   │   ├── OperationDate
│   │   │   : Fecha de operación de la factura
│   │   ├── ContractReference
│   │   │   : Referencia de contrato (opcional)
│   │   ├── ManagerUID
│   │   │   : UID del gestor (se sobreescribe con _credential)
│   │   ├── BillConcepts
│   │   │   : Lista de conceptos/líneas de factura
│   │   │   └── List<InstantBillConceptDTO>
│   │   │       ├── ConceptID
│   │   │       ├── Description
│   │   │       ├── Units
│   │   │       ├── Price
│   │   │       ├── VATID
│   │   │       ├── PercentDiscount
│   │   │       └── Order
│   │   ├── Bills
│   │   │   : Lista de vencimientos
│   │   │   └── List<InstantBillIssuedDTO>
│   │   │       ├── BillDate
│   │   │       ├── Amount
│   │   │       ├── BillStateType
│   │   │       ├── PaymentMethodType
│   │   │       ├── PaymentBankAccount
│   │   │       └── IBANSEPA
│   │   ├── Files
│   │   │   : Archivos adjuntos (se guardan aparte)
│   │   └── FilesDelete
│   │       : Archivos a eliminar
│   │
│   │
│   └── ExerciseDTE exerciseDTE
│       : Ejercicio fiscal actual
│
│       Campos importantes:
│       └── ExerciseID
│           : ID del ejercicio vigente
│
│       Se usa para:
│       └── Resolver la serie correcta al ejercicio actual
│

│   │
│   ├── _verifactuSignedCheckService
│   │       .ValidateVerifactuCheckAsync(CompanyUID)

│   │
│   │   Resultado:
│   │   └── verifactuCheck

│   │       ├── Reason (string)
│   │       │   : Motivo de bloqueo
│   │       └── CompanyNIF (string)
│   │           : NIF de la empresa
│   │
│   │
│   ├── CHECK: verifactuCheck.CanCreate == false
│   │   │
│   │   └── ❌ throw IESACommonException(verifactuCheck.Reason)
│   │       : La empresa no puede emitir facturas
│   │       : Ej: "Empresa bloqueada por Verifactu"
│   │
│   │
│   └── CHECK: verifactuCheck.CanCreate == true
│       │
│       └── STEP 1.1 → VALIDACIÓN TICKETBAI (TBAI)
│           │
│           ├── _verifactuSignedCheckService
│           │       .GetCheckSignedByCompanyUIDAsync(CompanyUID)
│           │
│           │   Resultado:
│           │   └── tbaiCheck
│           │       ├── SendToTaxAgency (bool)
│           │       │   : ¿Envía facturas a Hacienda?
│           │       └── UsesTBAI (bool)
│           │           : ¿Usa sistema TicketBAI?
│           │
│           │
│           ├── CONDICIÓN:
│           │   tbaiCheck.SendToTaxAgency == true
│           │   && tbaiCheck.UsesTBAI == true
│           │
│           │   : Solo aplica a País Vasco (Álava, Guipúzcoa, Vizcaya)
│           │
│           │
│           └── CHECK CERTIFICADO:
│               │
│               └── IsCertificateUpload(verifactuCheck.CompanyNIF)
│                   │
│                   │   ┌──────────────────────────────────────┐
│                   │   │ IsCertificateUpload() (privado)       │
│                   │   ├──────────────────────────────────────┤
│                   │   │ _certificatesClientService            │
│                   │   │   .CheckCertificateByNIFObject(nif)   │
│                   │   │                                       │
│                   │   │ Comprueba si existe certificado        │
│                   │   │ digital subido en Azure                │
│                   │   │                                       │
│                   │   │ cert != null                           │
│                   │   │ && cert.operationSuccess == true       │
│                   │   │   → return true                       │
│                   │   │                                       │
│                   │   │ else                                   │
│                   │   │   → return false                      │
│                   │   └──────────────────────────────────────┘
│                   │
│                   ├── false
│                   │   └── ❌ throw "No se dispone de certificado
│                   │              para envíos a TicketBai.
│                   │              NIF: {NIF}"
│                   │
│                   └── true
│                       └── ✅ continúa
│

├── STEP 2 → VALIDACIÓN DE BORRADOR (opcional)
│   │
│   ├── CHECK: instantInvoiceDTO.DraftInvoiceIssuedID != null
│   │
│   │   : La factura viene de un borrador previo
│   │
│   │   false
│   │   └── ✅ saltar paso (no hay borrador)

│   │   true

│   │
│   ├── _draftInvoiceIssuedService
│   │       .UpdateDraftInvoiceIssuedStatusAsync(DraftID)
│   │
│   │   : Marca el borrador como "facturado"
│   │   : Devuelve true si YA estaba facturado

│   │
│   │   Propósito:
│   │   └── Evitar crear facturas duplicadas desde el mismo borrador

│   │
│   ├── CHECK: invoiced == true
│   │   │
│   │   └── ❌ throw "La factura ya esta creada"
│   │       : El borrador ya fue convertido en factura antes
│   │       : Protección contra doble click / peticiones duplicadas
│   │
│   │
│   └── invoiced == false
│       └── ✅ borrador marcado como facturado, continúa

│   │

│   │           └── ... (nombre, dirección, etc.)
│   │
│   │
│   ├── CHECK: participantDTO == null
│   │   │
│   │   └── ❌ throw "Cliente no encontrado"
│   │
│   │
│   └── STEP 3.3 → VALIDAR DATOS DEL CLIENTE
│       │
│       └── Utilities.ValidateParticipant(participantDTO)

│           : Valida que el cliente tenga todos los datos obligatorios

│           ├── Nombre/Razón social
│           ├── NIF/CIF
│           ├── Dirección
│           ├── Código postal
│           └── Otros campos requeridos
│
│           CHECK: error.Length > 0
│           └── ❌ throw IESACommonException(errores acumulados)

├── STEP 4 → VALIDACIÓN Y RESOLUCIÓN DE SERIE
│   │
│   ├── STEP 4.1 → SERIE SELECCIONADA
│   │   │
│   │   └── CHECK: SerieID == 0 || SerieID == null
│   │       │
│   │       └── ❌ throw "Debes seleccionar una serie"
│   │
│   │
│   ├── STEP 4.2 → RESOLVER SERIE AL EJERCICIO ACTUAL
│   │   │
│   │   └── GetCurrentSerieToExerciseAsync(SerieID, exerciseDTE)
│   │
│   │       ┌──────────────────────────────────────────────────────┐
│   │       │ GetCurrentSerieToExerciseAsync() (privado)            │
│   │       ├──────────────────────────────────────────────────────┤
│   │       │                                                       │
│   │       │ _serieService.GetByIdAsync<SerieDTE>(serieID)         │
│   │       │   └── currentSerie                                   │
│   │       │                                                       │
│   │       │ CASO 1:                                               │
│   │       │   currentSerie.ExerciseID == exerciseDTE.ExerciseID   │
│   │       │   : La serie pertenece al ejercicio actual            │
│   │       │   └── return currentSerie (sin cambios)              │
│   │       │                                                       │
│   │       │ CASO 2:                                               │
│   │       │   currentSerie.ExerciseID != exerciseDTE.ExerciseID   │
│   │       │   : La serie pertenece a otro ejercicio               │
│   │       │   : Buscar serie equivalente en ejercicio actual      │
│   │       │   │                                                   │
│   │       │   ├── Filtro:                                         │
│   │       │   │   ├── ExerciseID = ejercicio actual               │
│   │       │   │   ├── Name = mismo nombre de serie                │
│   │       │   │   ├── IsActive = true                             │
│   │       │   │   └── Limit = 1                                  │
│   │       │   │                                                   │
│   │       │   ├── Encontrada                                      │
│   │       │   │   └── return targetSerie                         │
│   │       │   │                                                   │
│   │       │   └── No encontrada                                   │
│   │       │       └── ❌ throw "No existe una serie activa        │
│   │       │              '{nombre}' para el ejercicio actual"     │
│   │       └──────────────────────────────────────────────────────┘
│   │
│   │       Resultado:
│   │       └── currentSerie
│   │           ├── SerieID (del ejercicio actual)
│   │           ├── Name (ej: "A", "B", "R")
│   │           ├── Counter (último número usado)
│   │           └── ExerciseID
│   │
│   │       Actualiza DTO:
│   │       ├── instantInvoiceDTO.SerieID = currentSerie.SerieID
│   │       └── instantInvoiceDTO.Series = currentSerie.Name
│   │
│   │
│   └── STEP 4.3 → VALIDAR SERIE RESULTANTE
│       │
│       └── CHECK: instantInvoiceDTO.Series == null
│           │
│           └── ❌ throw "Debes seleccionar una serie"

├── STEP 7 → CONSTRUCCIÓN DE LA ENTIDAD FACTURA
│   │
│   ├── STEP 7.1 → SNAPSHOT DATOS PERSONALES CLIENTE
│   │   │
│   │   └── _invoiceIssuedPersonDataService
│   │           .PersistInvoiceIssuedPersonDataToParticipantAsync(
│   │               participantDTO)
│   │
│   │       : Guarda una "foto" de los datos del cliente
│   │       : en el momento exacto de la facturación
│   │       : Si el cliente cambia de dirección después,
│   │       : la factura conserva los datos originales
│   │
│   │       Resultado:
│   │       └── invoiceIssuedPersonDataDTE
│   │           ├── InvoiceIssuedPersonDataID
│   │           │   : 0 → datos nuevos (no existían antes)
│   │           │   : >0 → datos ya existentes (reutiliza)
│   │           └── InvoiceIssuedPersonData
│   │               : Objeto con nombre, NIF, dirección, etc.
│   │
│   │
│   ├── STEP 7.2 → MAPEO DTO → DTE
│   │   │
│   │   └── invoiceIssuedDTE = _mapper.Map<InvoiceIssuedDTE>(
│   │                              instantInvoiceDTO)
│   │
│   │       : AutoMapper convierte el DTO de entrada
│   │       : a la entidad de base de datos
│   │
│   │       Campos mapeados automáticamente:
│   │       ├── CompanyUID
│   │       ├── ParticipantUID
│   │       ├── SerieID
│   │       ├── Series
│   │       ├── OperationDate
│   │       ├── ContractReference
│   │       └── ... otros campos del DTO
│   │
│   │
│   ├── STEP 7.3 → ASIGNACIÓN DE CAMPOS ADICIONALES
│   │   │
│   │   ├── invoiceIssuedDTE.ManagerUID = _credential.uidGestor
│   │   │   : Gestor autenticado (del token JWT)
│   │   │   : Sobreescribe cualquier valor del DTO
│   │   │
│   │   ├── invoiceIssuedDTE.InvoicingDate = DateTime.Now
│   │   │   : Fecha de emisión = momento actual del servidor
│   │   │   : No se puede manipular desde el cliente
│   │   │
│   │   └── invoiceIssuedDTE.InvoiceType = TpGFCloudInvoiceType.Issued
│   │       : Tipo = Emitida (valor fijo)
│   │       : Las rectificativas se crean por otro método
│   │
│   │
│   ├── STEP 7.4 → VINCULAR DATOS PERSONALES
│   │   │
│   │   ├── CASO A: InvoiceIssuedPersonDataID == 0
│   │   │   : Datos personales NUEVOS (primera vez que se factura
│   │   │   : a este cliente con estos datos exactos)
│   │   │   │
│   │   │   └── invoiceIssuedDTE.invoiceIssuedPersonData =
│   │   │           invoiceIssuedPersonDataDTE.InvoiceIssuedPersonData
│   │   │       : Asigna como propiedad de navegación
│   │   │       : EF Core creará el registro automáticamente
│   │   │       : al hacer PersistAsync de la factura
│   │   │
│   │   │
│   │   └── CASO B: InvoiceIssuedPersonDataID > 0
│   │       : Datos personales EXISTENTES (ya se facturó antes
│   │       : con los mismos datos, reutiliza el registro)
│   │       │
│   │       └── invoiceIssuedDTE.InvoiceIssuedPersonDataID =
│   │               invoiceIssuedPersonDataDTE.InvoiceIssuedPersonDataID
│   │           : Asigna solo la FK
│   │           : No crea registro nuevo
│   │
│   │
│   └── STEP 7.5 → VINCULAR BORRADOR (opcional)
│       │
│       └── CHECK: instantInvoiceDTO.DraftInvoiceIssuedID != null
│           │
│           └── invoiceIssuedDTE.DraftInvoiceIssuedID =
│                   instantInvoiceDTO.DraftInvoiceIssuedID
│               : Mantiene trazabilidad borrador → factura

├── STEP 8 → CÁLCULO DE IMPORTES
│   │
│   ├── STEP 8.1 → CALCULAR CONCEPTOS
│   │   │
│   │   └── _billConceptService
│   │           .CalculateInstantBillConceptAmountsAsync(
│   │               companyDTE,
│   │               participantDTO,
│   │               instantInvoiceDTO.BillConcepts)
│   │
│   │       : Calcula para CADA línea de factura:
│   │
│   │       ┌────────────────────────────────────────────────┐
│   │       │ Por cada concepto:                              │
│   │       │                                                 │
│   │       │ 1. Obtiene impuesto IVA del concepto (VATID)    │
│   │       │ 2. Obtiene retención empresa (TaxUID)           │
│   │       │                                                 │
│   │       │ 3. BaseAmount =                                 │
│   │       │    (Units × Price) × (1 - %Descuento/100)       │
│   │       │                                                 │
│   │       │ 4. VATAmount =                                  │
│   │       │    BaseAmount × VATPercent / 100                 │
│   │       │                                                 │
│   │       │ 5. WithholdingAmount =                          │
│   │       │    BaseAmount × WithholdingPercent / 100         │
│   │       │    (solo si participantDTO.IsWithHoldingTax)     │
│   │       │                                                 │
│   │       │ 6. Amount =                                     │
│   │       │    BaseAmount + VATAmount - WithholdingAmount    │
│   │       │                                                 │
│   │       │ 7. Redondeo a 2 decimales                       │
│   │       └────────────────────────────────────────────────┘
│   │
│   │       Resultado:
│   │       └── List<BillConceptDTE> billConceptDTEs
│   │           : Lista con importes calculados
│   │
│   │
│   ├── STEP 8.2 → CALCULAR TOTAL FACTURA
│   │   │
│   │   └── invoiceIssuedDTE.TotalAmount =
│   │           billConceptDTEs.Sum(d => d.Amount)
│   │
│   │       : Suma de todos los importes finales de cada línea
│   │       : Incluye IVA y descuenta retenciones
│   │
│   │       Ej:
│   │       ├── Línea 1: Amount = 605.00  (500 base + 105 IVA)
│   │       ├── Línea 2: Amount = 605.00  (500 base + 105 IVA)
│   │       └── TotalAmount = 1210.00
│   │
│   │
│   ├── STEP 8.3 → VALIDAR TOTAL vs VENCIMIENTOS
│   │   │
│   │   └── CHECK:
│   │       invoiceIssuedDTE.TotalAmount !=
│   │           instantInvoiceDTO.Bills.Sum(b => b.Amount)
│   │       │
│   │       └── ❌ throw "El importe de la factura no coincide
│   │                     con el importe de los vencimientos."
│   │
│   │       Ej incorrecto:
│   │       ├── TotalAmount = 1210.00
│   │       ├── Vencimiento 1: 600.00
│   │       ├── Vencimiento 2: 600.00
│   │       └── Suma vencimientos: 1200.00 ≠ 1210.00 → ❌
│   │
│   │       Ej correcto:
│   │       ├── TotalAmount = 1210.00
│   │       ├── Vencimiento 1: 605.00
│   │       ├── Vencimiento 2: 605.00
│   │       └── Suma vencimientos: 1210.00 == 1210.00 → ✅
│   │
│   │
│   └── STEP 8.4 → VALIDAR TOTAL NO NEGATIVO
│       │
│       └── CHECK: invoiceIssuedDTE.TotalAmount < 0
│           │
│           └── ❌ throw "El importe total de la factura
│                         no puede ser negativo."
│               : Las facturas con importe negativo son
│               : rectificativas y se crean por otro flujo
│               : (CreateRectifyInvoiceAsync)
│

├── STEP 9 → PERSISTENCIA (dentro de la transacción)
