DESCRIPCIÓN DEL REDMINE
Se crea una nueva tabla RecurringInvoiceItemAdditionalData que guarda datos exclusivos de los conceptos de facturas periódicas a inquilinos (IsServiceFee, IsBillable, IsRentReview). Estos campos por defecto son false y no puede fallar si no existen registros en la tabla.
La tabla se alimenta cuando se crea/edita una factura periódica (POST/PUT de RecurringInvoice). En los GET, se devuelven los 3 campos en el objeto RecurringInvoiceItemDTO.

using IESA.TC.NetCore.Utilities.Models;
using Microsoft.Extensions.Logging;

namespace IESA.TC.Invoices.V2.Infrastructure.RecurringInvoiceItemAdditionalData
{
    public class RecurringInvoiceItemAdditionalDataService : BaseService<RecurringInvoiceItemAdditionalDataEntity, RecurringInvoiceItemAdditionalDataDTE>, IRecurringInvoiceItemAdditionalDataService
    {
        private readonly IRecurringInvoiceItemAdditionalDataRepository _repository;
        private readonly TCContext _credential;

        public RecurringInvoiceItemAdditionalDataService(
            ILogger<RecurringInvoiceItemAdditionalDataService> pLogger,
            IMapper pMapper,
            DBContext pDbContext,
            TCContext credential,
            IRecurringInvoiceItemAdditionalDataRepository repository)
            : base(pMapper, pLogger, pDbContext, repository)
        {
            _repository = repository;
            _credential = credential;
        }

        public async Task<RecurringInvoiceItemAdditionalDataDTE?> GetByRecurringInvoiceItemIDAsync(long recurringInvoiceItemID)
        {
            _logger.LogInformation("{method} --> Start", nameof(GetByRecurringInvoiceItemIDAsync));

            var result = await _repository.GetByRecurringInvoiceItemIDAsync(recurringInvoiceItemID);

            _logger.LogInformation("{method} --> End", nameof(GetByRecurringInvoiceItemIDAsync));

            return result;
        }

        public async Task SaveRecurringInvoiceItemAdditionalDataAsync(RecurringInvoiceItemAdditionalDataDTE additionalDataDTE)
        {
            _logger.LogInformation("{method} --> Start", nameof(SaveRecurringInvoiceItemAdditionalDataAsync));

            additionalDataDTE.ManagerUID = _credential.uidGestor;
            await PersistAsync(additionalDataDTE);

            _logger.LogInformation("{method} --> End", nameof(SaveRecurringInvoiceItemAdditionalDataAsync));
        }
    }
}

EXPLICACIÓN:
•	Hereda de BaseService: Da acceso a PersistAsync (que gestiona INSERT o UPDATE según si el ID es 0 o no), GetByIdAsync, DeactivateAsync, etc.
•	_credential.uidGestor: Es el UID del gestor/tenant del usuario autenticado. Se obtiene del token JWT que viene en la request HTTP.
•	GetByRecurringInvoiceItemIDAsync: Simplemente delega al repositorio. La capa de servicio existe para añadir lógica de negocio si fuera necesario en el futuro.
•	SaveRecurringInvoiceItemAdditionalDataAsync: ANTES de persistir, asigna el ManagerUID. Sin esto, el INSERT falla porque ManagerUID es NOT NULL en la BD. PersistAsync del BaseService:
•	Si RecurringInvoiceItemAdditionalDataID == 0 → hace INSERT (y el framework genera el UID automáticamente por el atributo [Uid]).
•	Si RecurringInvoiceItemAdditionalDataID != 0 → hace UPDATE.
•	También gestiona automáticamente CreatedAt/ModifiedAt y CreatedByUserUID/ModifiedByUserUID.

2.	Se valida participante, empresa, NIF, dirección.
3.	Se calcula importes de los items.
4.	Se persiste RecurringInvoice (con ManagerUID asignado).
5.	Se persisten los RecurringInvoiceItems.
6.	Para cada item: se crea un registro en RecurringInvoiceItemAdditionalData con los 3 campos booleanos.
7.	Se procesan los items a eliminar.
GET (obtener facturas periódicas para facturación):
1.	Se obtienen las facturas periódicas del repositorio.
2.	Para cada factura, se obtienen sus items.
3.	Para cada item, se busca su registro adicional y se enriquece el DTO con isServiceFee, isBillable, isRentReview (default false si no existe).
4.	Se devuelve al frontend con los campos completos.
PUT (editar factura periódica existente):
1.	Mismo flujo que POST pero RecurringInvoiceID != 0.
2.	Se obtiene el RecurringInvoice existente de BD.
3.	Al guardar AdditionalData: busca el existente (UPDATE) en vez de crear nuevo (INSERT).

PROBLEMA ENCONTRADO Y SOLUCIONADO
Error: "Cannot insert the value NULL into column 'ManagerUID', table 'invoices-v2-test.dbo.RecurringInvoice'"
Causa: AutoMapper copiaba los campos del DTO al DTE, pero el DTO no tiene ManagerUID. El DTE quedaba con ManagerUID = null (string.Empty se sobreescribía por el mapper).
Solución: Añadir recurringInvoiceDTE.ManagerUID = _credential.uidGestor; DESPUÉS del mapper.
